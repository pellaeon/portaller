How to setup

There are two ways of how to use Portaller. If you want to unblock content for one device only (e.g. iPhone or SmartTV) — find that device in the left sidebar and set it up as described.

But if you want to use it on many devices inside your home network at the same time (e.g. listen to Pandora on iPhone and watch Netflix on TV) — best way is to write Portaller DNS address to the settings of your router. It will push the right DNS settings to any devices connected to it.

Windows 7

  • Click the "Network" icon in the status bar and open "Network center";
  • Select a current active connection;
  • Click "Options" in the window appeared;
  • Select "IPv4" row and choose "Options";
  • Put "107.170.15.247" as the primary DNS and "77.88.8.8" as the secondary one in the "Preferred DNS" section.


    • Mac OS X

  • Open "Network" section;
  • Select active connection and click "Advanced";
  • Open "DNS" section;
  • Click "Add" and put "107.170.15.247" as primary DNS and "77.88.8.8" as secondary.


    • Mac OS X Manual

    If you're on MacOS and don't want to send all DNS queries to our server, but only geo-locked domains, you can set specific DNS resolver to use for each domain, similar to dnsmasq on Linux detailed below.

  • $ sudo mkdir -p /etc/resolver/
  • Create a text file named after the domain you want to specify, for example pandora.com, the DNS settings will also be applied to its subdomains: $ sudo touch /etc/resolver/pandora.com
  • Inside the file put in Portaller DNS setting: 
    nameserver 107.170.15.247

    Mikrotik

    If you use Mikrotik as a router, you can set it up to use Portaller DNS only for specified domains. To do this create a few rules for every domain (for Pandora as an example):

    /ip firewall layer7-protocol add name=pandora.com regexp=pandora.com

    /ip firewall mangle add action=mark-connection chain=prerouting dst-address=192.168.88.1 dst-port=53 layer7-protocol=pandora.com new-connection-mark=forwarded-dns protocol=udp

    /ip firewall nat add action=dst-nat chain=dstnat comment="portaller.com dns proxy" connection-mark=forwarded-dns to-addresses=107.170.15.247

    Where 192.168.88.1 Mikrotik IP, and 107.170.15.247 Portaller IP.
